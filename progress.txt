18) Evaluation and observability layer added (March 10, 2026).
    Inspired by analysis of Corti's agentic framework gaps.

    Changes:
    - app/guardrails.py: added classify_failure() with 6 failure categories.
      Maps guardrail results to one human-readable label (plan_generation_failure,
      routing_error, tool_skip, citation_missing, source_missing, pass).
    - app/guardrails.py: added check_groundedness() — LLM-as-judge that compares
      the final answer against tool outputs. Returns grounded/ungrounded/partial.
      Also added extract_tool_outputs() helper.
    - app/state.py: added planner_reasoning (str) and groundedness_check (dict)
      to AgentState.
    - app/planner_agent.py: captures raw LLM output (or fast-path label) before
      JSON parsing, stores in state["planner_reasoning"].
    - app/config.py: added GROUNDEDNESS_MODEL, GROUNDEDNESS_NUM_CTX (1024),
      GROUNDEDNESS_TEMPERATURE (0.1).
    - app/graph.py: creates groundedness LLM instance. Defaults to reusing the
      answerer model to avoid VRAM swap on low-memory hardware. Returns it
      alongside the app from build_app().
    - app/run_cli.py: runs groundedness check after graph completes (avoids
      token leakage into streamed answer). Writes judge result as separate
      JSONL trace entry. Prints verdict only for non-grounded cases.
    - app/eval_runner.py: adds failure_mode, planner_reasoning, and
      groundedness_check to each question result.
    - tests/test_eval_improvements.py: 19 new tests (failure classification,
      tool output extraction, groundedness check with edge cases).
    - first_principle_pipeline.md: created. Explains full pipeline from first
      principles with decision rationale.
    - ARCHITECTURE.md: updated with evaluation/observability sections.

    Design decisions:
    - Groundedness judge runs AFTER graph (not inside answerer node) because
      LangGraph streaming picks up judge tokens and displays them to user.
    - Judge model defaults to answerer model to reuse VRAM. Override via
      GROUNDEDNESS_MODEL env var on better hardware.
    - Judge context window is 1024 (answers + tool outputs < 750 words).
    - Failure classification uses priority ordering (plan > routing > tools > format).
